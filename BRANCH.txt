1. STRUCTURE DES BRANCHES

Le projet utilise deux branches principales :

┌─────────────────────────────────────────────────────────────────┐
│  MAIN (branche principale)                                     │
│  ├── Code stable et testé                                     │
│  ├── Version de production                                    │

└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │
                              │  Pull Request
                              │
┌─────────────────────────────────────────────────────────────────┐
│  CODING (branche de développement)                             │
│  ├── Code en cours de développement                            │

2. COMMANDES DE BASE
================================================================

📌 VÉRIFIER LA BRANCHE ACTUELLE
┌─────────────────────────────────────────────────────────────────┐
│  git branch                                                    │
│  git branch --show-current                                     │
│  # Affiche la branche sur laquelle vous êtes                   │
└─────────────────────────────────────────────────────────────────┘

📌 VOIR TOUTES LES BRANCHES
┌─────────────────────────────────────────────────────────────────┐
│  git branch -a                                                 │
│  # Affiche toutes les branches (locales et distantes)          │
│  # * coding  → branche actuelle                                │
│  #   main    → branche principale                              │
│  # remotes/origin/* → branches distantes                       │
└─────────────────────────────────────────────────────────────────┘

📌 CHANGER DE BRANCHE
┌─────────────────────────────────────────────────────────────────┐
│  git checkout main          # Aller sur main                   │
│  git checkout coding        # Aller sur coding                 │
│  git checkout -b nouvelle   # Créer et aller sur nouvelle      │
└─────────────────────────────────────────────────────────────────┘

3. TRAVAILLER SUR LA BRANCHE CODING
================================================================

📌 ÉTAPE 1 : SE METTRE SUR CODING
┌─────────────────────────────────────────────────────────────────┐
│  git checkout coding                                            │
│  # Vérifier : git branch --show-current                        │
│  # Doit afficher : coding                                      │
└─────────────────────────────────────────────────────────────────┘

📌 ÉTAPE 2 : RÉCUPÉRER LES DERNIÈRES MODIFICATIONS
┌─────────────────────────────────────────────────────────────────┐
│  git pull origin coding                                        │
│  # Récupère les modifications distantes sur coding             │
└─────────────────────────────────────────────────────────────────┘

📌 ÉTAPE 3 : FAIRE DES MODIFICATIONS
┌─────────────────────────────────────────────────────────────────┐
│  # Modifier vos fichiers (ex: Restaurant_GUI.py)              │
│  # Ajouter de nouvelles fonctionnalités                        │
│  # Corriger des bugs                                           │
└─────────────────────────────────────────────────────────────────┘

📌 ÉTAPE 4 : AJOUTER LES MODIFICATIONS
┌─────────────────────────────────────────────────────────────────┐
│  git add .                                                     │
│  # Ajoute tous les fichiers modifiés                           │
│  # OU                                                          │
│  git add nom_du_fichier.py                                     │
│  # Ajoute un fichier spécifique                                │
└─────────────────────────────────────────────────────────────────┘

📌 ÉTAPE 5 : COMMITTER LES MODIFICATIONS
┌─────────────────────────────────────────────────────────────────┐

│  - Dossier recus/ créé automatiquement"                       │

📌 ÉTAPE 6 : POUSSER SUR GITHUB
┌─────────────────────────────────────────────────────────────────┐

│  # OU simplifié :                                              │
│  git push                                                      │

4. SYNCHRONISER AVEC LA BRANCHE MAIN
================================================================

📌 RÉCUPÉRER LES MODIFICATIONS DE MAIN
┌─────────────────────────────────────────────────────────────────┐
│  # 1. Aller sur main                                           │
│  git checkout main                                             │
│                                                                 │
│  # 2. Récupérer les modifications                              │
│  git pull origin main                                          │
│                                                                 │
│  # 3. Revenir sur coding                                       │
│  git checkout coding                                           │
│                                                                 │
│  # 4. Fusionner les changements de main dans coding           │
│  git merge main                                                │
│                                                                 │
│  # 5. Résoudre les conflits si nécessaire                      │
│  # (voir section Conflits)                                    │
│                                                                 │
│  # 6. Pousser les modifications                                │
│  git push origin coding                                        │
└─────────────────────────────────────────────────────────────────┘

📌 FUSIONNER CODING DANS MAIN (Pull Request)
┌─────────────────────────────────────────────────────────────────┐
│  # MÉTHODE 1 : Via GitHub CLI                                  │
│  gh pr create --base main --head coding                        │
│                                                                 │
│  # MÉTHODE 2 : Via VS Code                                     │
│  1. Ctrl+Shift+P                                               │
│  2. GitHub: Create Pull Request                                │
│  3. Sélectionner base: main, compare: coding                   │
│                                                                 │
│  # MÉTHODE 3 : Via GitHub web                                  │
│  1. Aller sur le dépôt GitHub                                  │
│  2. Cliquer sur "Pull requests"                               │
│  3. Cliquer sur "New pull request"                            │
│  4. Sélectionner base: main, compare: coding                   │
│  5. Créer et fusionner                                         │
└─────────────────────────────────────────────────────────────────┘

5. GÉRER LES CONFLITS
================================================================

📌 IDENTIFIER UN CONFLIT
┌─────────────────────────────────────────────────────────────────┐
│  git status                                                    │
│  # Affiche : both modified: nom_du_fichier.py                  │
└─────────────────────────────────────────────────────────────────┘

📌 RÉSOUDRE UN CONFLIT
┌─────────────────────────────────────────────────────────────────┐
│  # 1. Ouvrir le fichier en conflit                            │
│  # VS Code affiche les sections :                              │
│  # <<<<<<< HEAD     → vos modifications                       │
│  # =======           → séparateur                              │
│  # >>>>>>> main     → modifications de main                    │
│                                                                 │
│  # 2. Choisir la version à garder                              │
│  # - Garder les deux                                           │
│  # - Garder l'une des deux                                     │
│  # - Fusionner manuellement                                    │
│                                                                 │
│  # 3. Supprimer les marqueurs (<<<<<<<, =======, >>>>>>>)     │
│                                                                 │
│  # 4. Ajouter le fichier résolu                                │
│  git add nom_du_fichier.py                                     │
│                                                                 │
│  # 5. Continuer le merge                                       │
│  git merge --continue                                          │
│                                                                 │
│  # 6. OU annuler le merge                                      │
│  git merge --abort                                             │
└─────────────────────────────────────────────────────────────────┘

6. COMMANDES UTILES AU QUOTIDIEN
================================================================

📌 VÉRIFICATIONS
┌─────────────────────────────────────────────────────────────────┐

│  git log --oneline -5 # Voir les 5 derniers commits            │

┌─────────────────────────────────────────────────────────────────┐

│  git restore .                  # Annuler toutes les modifs     │

│  git reset --soft HEAD~1        # Annuler le dernier commit    │
│  git reset --hard HEAD~1        # Annuler commit et modifs     │
└─────────────────────────────────────────────────────────────────┘

📌 BRANCHES
┌─────────────────────────────────────────────────────────────────┐
│  git branch          # Voir les branches locales               │
│  git branch -a       # Voir toutes les branches                │
│  git branch -d nom   # Supprimer une branche locale            │
│  git push origin --delete nom  # Supprimer une branche distante│

📌 PULL REQUESTS (GitHub CLI)
┌─────────────────────────────────────────────────────────────────┐

│  gh pr list                              # Voir les PRs        │
